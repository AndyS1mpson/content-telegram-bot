# Команды бота

Все команды доступны **только владельцу** (`bot_owner_id` в [config.yaml](../config.example.yaml)). Посторонний пользователь получит `you do not have access`.

## /start

Без аргументов. Приветствие и список команд. Полезно сразу после деплоя, чтобы проверить что бот жив и owner_id настроен правильно.

## /collect \<тема\>

Запускает парсинг Pinterest по заданной теме. `<тема>` — всё после команды, пробелы внутри разрешены.

```
/collect nature 4k wallpaper
/collect минимализм обои
/collect cyberpunk city
```

**Что делает:**
1. Логинится на Pinterest как указанный в конфиге аккаунт.
2. Переходит на `https://ru.pinterest.com/search/pins/?q=<encoded>`.
3. Забирает до 30 картинок с первой страницы выдачи.
4. Конвертирует URL превьюшек в URL оригиналов (`i.pinimg.com/originals/…`).
5. Сохраняет в БД со статусом `New`. Дубликаты по `(id, channel)` пропускаются — безопасно запускать повторно.

**Типичное время:** 15–40 секунд (логин + отрисовка страницы).

**Частые проблемы:**
- `parse error: sign in: ...` — Pinterest запросил капчу или заблокировал логин. Смените пароль, отключите 2FA или используйте другой аккаунт.
- `parse error: wait for images locator` — страница не отрендерила картинки (сеть, JS, смена вёрстки Pinterest).

## /view \<тема\>

Показывает следующий неотсмотренный пин (статус `New`) по указанной теме. Сообщение приходит с фото/видео и тремя кнопками:

| Кнопка | Действие | Статус пина после |
|---|---|---|
| ❤️ В публикацию | Отметить для `/publish` | `Selected` |
| 👎 Отклонить | Навсегда пропустить | `Viewed` |
| ⏭ Пропустить | Навсегда пропустить | `Viewed` |

После нажатия бот:
- Снимает кнопки у предыдущего сообщения и меняет caption на `Тема: X ✅ В публикации` / `❌ Отклонён` / `⏭ Пропущен`.
- Сразу присылает следующий пин.

**Если `New` закончились** — бот пишет «Новых пинов нет, собираю ещё…» и автоматически зовёт `/collect <та же тема>`, после чего показывает следующий. Если Pinterest вернул тот же набор (все — дубликаты уже виденных), то бот ответит «По теме "X" больше нечего показать» — это сигнал, что тему стоит разнообразить или использовать `LikePins` для прогрева ленты.

Команда `/view` без дальнейших нажатий сама по себе переводит показанный пин в `Viewed` атомарно. Это нужно, чтобы один и тот же пин не показывался дважды при двойном нажатии команды.

## /publish \<тема\>

Публикует все отобранные (`Selected`) пины по указанной теме в Telegram-канал, указанный в `telegram_chat_id` первого аккаунта из конфига.

**Что делает:**
1. Достаёт все пины со статусом `Selected` по `(channel, query)`.
2. Разбивает на пачки по 10 (лимит Telegram `sendMediaGroup`).
3. Для каждой пачки формирует `media_group` из `InputMediaPhoto` / `InputMediaVideo` с URL пина — Telegram скачивает файлы сам.
4. После успешной отправки всех пачек проставляет `Posted`.

Бот должен быть **админом** канала (с правом `Post Messages`), иначе Telegram вернёт `chat not found` или `Forbidden: bot is not a member`.

Если отправка части пачек прошла успешно, а на следующей упала — уже отправленные пины помечаются `Posted`, остальные остаются `Selected` (можно повторить команду).

## Незарегистрированные команды

Любая неизвестная команда вернёт `Unknown command. Try /start`. Сообщения без команды игнорируются.

## Что видят пользователи не-владельцы

Любые команды → `you do not have access`. Никаких утечек ни темы, ни ID пинов, ни данных канала.

## Примеры полного сценария

```
/start
→ Привет! Я собираю обои с Pinterest ...

/collect минимализм
→ Собираю пины по теме: минимализм…
→ Готово. Смотри: /view минимализм

/view минимализм
→ [фото1 с кнопками, caption "Тема: минимализм  Осталось неотсмотренных: 29"]
❤️ → [фото1 меняется на "Тема: минимализм ✅ В публикации", сразу приходит фото2]
👎 → [фото2 "❌ Отклонён", приходит фото3]
...
(отобрали 12 штук)

/publish минимализм
→ [в канале: media group 10 картинок, потом ещё 2]
→ Опубликовано 12 пинов по теме "минимализм".
```
